Escopo no JavaScript

Na programação, o escopo refere-se à visibilidade e acessibilidade de variáveis, funções e objetos durante o tempo de execução. Simplificando, o escopo diz se você pode usar variáveis, funções ou objetos específicos no seu código em um local específico ou não.

Explicação do ChatGPT

Escopo no JavaScript é o "limite" onde as variáveis e funções podem ser acessadas. Pense no escopo como o "ambiente" onde uma variável "vive" e está disponível para uso. Existem dois tipos principais de escopo:

  • Escopo Global:Variáveis declaradas fora de qualquer função. Elas ficam disponíveis em qualquer parte do código.
  • Escopo Local (ou de Função):Variáveis declaradas dentro de uma função. Elas só podem ser acessadas dentro dessa função.

Além disso, há o escopo de bloco, criado quando usamos { } com let e const (em if, for, etc.). Variáveis dentro desses blocos só podem ser acessadas ali.

  • Global: acessível de qualquer lugar
  • Local/Função: acessível só dentro da função.
  • Bloco: acessível apenas no bloco { }.

Esses "limites" ajudam a controlar onde suas variáveis podem ser vistas e usadas, evitando erros e conflitos de nomes.
